Managed add-ons - The descriptor file

The descriptor contains information on all packages that you offer on your server. You can change the file at any time to introduce new packages, newer versions of existing packages or to update metadata.
All you have to do is to submit the URL to the descriptor once to the openBVE project. After that, it will be crawled automatically at least once every 48 hours in order to collect any changes you made to it.
Please find a list of things you can do in your descriptor below, in particular a list of all available metadata

The [package] section

[package] This starts a specific version of a package.
name The name of the package. Names must be unique and sufficiently self-descriptive. A simple A is not enough and will be rejected.
version The version number of the package. Please find out more about version numbers here.
size The exact size of the archive file in bytes.
md5 The MD5 of the archive. Considering using this tool if you do not know how to compute the MD5.
sources A comma-separated list of URLs where the package can be downloaded from. These must be direct links.
dependencies A comma-separated list of dependencies. A dependency has the form package[version]. This means your package will depend on the specified package of the specified version or any later version.
suggestions A comma-separated list of suggestions. A suggestion has the form package[version]. This means your package will suggest the specified package of the specified version or any later version.
entry The subdirectory within the package where openBVE starts looking for routes or trains. In case your route uses the Railway (Route, Object, Sound) structure, you must enter Railway\Route.
type The type of the package. Must be either route, train, library or shared library.
author The name of the author who created the package.
caption[code] A short caption that appears in list boxes.
description[code] A more detailed description of the package.
country[code] The country featured in the route, or to which the train can be attributed to. Should be left out if the route or train cannot be attributed to any particular country.
city[code] The city featured in the route, or to which the train can be attributed to. Should be left out if the add-on cannot be attributed to any particular city.
operator[code] The operator that runs the route or train. Should be left out if the route or train cannot be attributed to any particular operator.
fictional Should be set to true for fictional routes and trains.
screenshots A comma-separated list of URLs where screenshots of the add-on are stored. You should also provide thumbnails.
thumbnails A comma-separated list of URLs where thumbnails are stored (in the same order as screenshots). Thumbnails are smaller versions of screenshots for faster loading.
links A comma-separated list of URLs that provide links to manuals or the homepage of the add-on, and so on. You should also provide labels.
labels[code] A comma-separated list of labels for the links (in the same order as links).
Please note that none of the text fields or screenshots may include copyright notices.

Language codes

Some metadata such as country, city and description can be offered in multiple languages. You do so by including a language code in brackets, for example:
country[en-US] = Japan
country[ja-JP] = 日本
Please always include an English version of the metadata. For a full list of codes, see here.

The [links] section

If you want to organize yourself with others in your own communities, it is also possible that you include a [links] section. Instead of having to submit their descriptors to the openBVE project directly, other developers can also submit them to you. You then list those URLs in the [links] section, each URL on a separate line. Once the crawler reads your descriptor, all descriptors you link to will be automatically added to the crawler's database.
