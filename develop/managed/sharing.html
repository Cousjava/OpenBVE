Managed add-ons - Creating reusable packages with shared libraries

In Splitting your add-on into multiple packages, you have learned how to split your add-on into multiple packages and how to reference one package from within another.
There is nothing preventing you from referencing packages created by other authors, but there is no guarantee that the internal structure of a package stays the same in future versions. This means that if you referenced a file in someone else's package, that file might not be there in future versions, thus braking your add-on.
For this reason, you should generally only reference packages you have created yourself. But there are also shared libraries - packages specifically intended for others to reuse.

What is a shared library?

Basically, a shared library is the same as a regular library package, but comes with the promise that changes in future versions will retain backward compatibility on the file level.
This means that in a shared library, no file must ever be removed, renamed or suddenly depict something else.
The benefit - consider a package of signals - is that improvements can be made at any time by the author of the shared library, and those improvements will affect all add-ons that reference the shared library. There are lots of candidates for shared libraries: rails, signals, beacons, stations, buildings, background images, trees, train plugins, and many more.
Of course, if you primarily want to offer objects such as buildings, you might want to only give the backward compatibility promise for the object files themselves, not for the textures used by them. In this case, include a README file that makes crystal clear which files are covered by the backward compatibility promise and which not. For example, give the backward compatibility promise to all objects, or all textures, or to no sounds.
Please note that in case you do break backward compatibility in your shared library, the specific version that breaks backward compatibility may be blocked permanently and an older version may be restored.
