Managed add-ons - Version numbers

Version numbers are an essential component in managed add-ons because the idea is that a newer version can always supersede an older version. In order for the game to be able to check version numbers against each other, they must adhere to a certain format:
1.2.3a
A version number is comprised of a group of integers separated by periods. At the end, an optional textual suffix may follow.
Here is a list of example version numbers sorted from smallest to largest:
  • 0.9
  • 1.0
  • 1.1
  • 1.1.3
  • 1.1.6
  • 1.1.10
  • 1.1.15
  • 1.2
  • 1.2a
  • 1.2b
  • 1.2c
  • 1.3
  • 2.0

Version number policy

Works in progress should carry version numbers less than 1.0, and finished products version numbers starting with 1.0. There are two criteria for whether an add-on should be considered a finished product: quality and internal consistency.
Quality is more of a subjective criterion, but as a general rule, routes should not contain just skeleton scenery, and trains should have a cab that is not hand-drawn.
Internal consistency, in case of a route, means that the route should not contain highly detailed scenery in one place and skeleton scenery somewhere else.
Please note that users have the option to filter out works in progress when deciding to download add-ons. This is solely based on the version number, so please respect these criteria when selecting a version number for your add-on.
